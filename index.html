About Rust-WebSocket

Rust-WebSocket is a WebSocket (RFC6455) library written in Rust.

Rust-WebSocket attempts to provide a framework for WebSocket connections (both clients and servers). The library is currently in an experimental state, but can work as a simple WebSocket server or client, with more functionality to come. There is some documentation, but no testing code at the moment, however that is being worked on.

The project can be found on GitHub and on crates.io.

Installation

To add a library release version from crates.io to a Cargo project, add this to the [dependencies] section of your Cargo.toml:

websocket = "~0.6.0"

To add the library's Git repository to a Cargo project, add this to your Cargo.toml:

[dependencies.websocket]

git = "https://github.com/cyderize/rust-websocket.git"

And add the line extern crate websocket; to your main.rs or lib.rs.

Current State

At the moment, the library provides both client and server WebSocket implementations, however these have not been tested thoroughly. There is support for handling WebSocket requests and responses, as well as fragmented messages. Secure WebSocket connections can be used though rust-openssl. Simply pass an SslStream to the WebSocketClient::new() function and use the WebSocketClient as normal.

There are some unit tests, but more are required (as well as integration test). Eventually, proper compliance with RFC6455 should be tested.
