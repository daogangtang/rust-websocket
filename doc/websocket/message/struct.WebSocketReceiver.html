Represents a WebSocket receiver which can receive data from the remote endpoint. All methods are task blocking (but not stream blocking, so you can send and receive concurrently). A WebSocketReceiver can be captured into another task for concurrency

use websocket::message::WebSocketMessage;
 
let receiver = client.receiver();
 
Thread::spawn(move || {
    // Will continuously try to receive messages
    for message in receiver.incoming() {
        match message {
            // Match the result
            Ok(message) => {
                // Match the type of message
                match message {
                    WebSocketMessage::Text(data) => {
                        println!("{}", data);
                    }
                    WebSocketMessage::Binary(data) => {
                        // ...
                    }
                    // ...
                    _ => { }
                }
            }
            Err(e) => { /* Could not receive the message */ }
        }
    }
}).detach();

fn receive_message(&mut self) -> IoResult<WebSocketMessage>

Wait for and accept a message (subjected to the underlying stream timeout). If the received message is fragmented, this function will not return until the final fragment has been received. If a control frame is received interleaved within a fragmented message, The control frame will be returned first, and the message will be returned on the next call to the function (or later if more control frames are received).
